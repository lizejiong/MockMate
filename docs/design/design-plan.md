# AI 面试教练设计规划

## 设计定位

专业、有力量、但不冰冷。界面应像一个可靠的训练工作台，而不是营销页或题库 App。

## 主题

App 跟随系统浅色 / 深色主题。深色用于沉浸式面试和高专注场景，浅色用于配置、历史和阅读型信息。

## 字体与排版

- 首页和关键流程使用明确的大标题。
- 工作台、报告、点评卡片使用紧凑层级，方便扫读。
- 正文优先 16px，保证移动端阅读舒适。
- 标签、状态、计数器使用 13px 左右的小字号。

## 签名元素

AI 消息左侧使用亮靛蓝竖线，既作为视觉识别，也提示内容来自 AI 侧。

## 点评分区卡片

每轮点评拆成固定区域：

- 回答亮点
- 存在问题
- 改进方向
- 参考回答结构
- 追问决策
- 当前评分

卡片用于承载结构化信息，不用于堆砌装饰。

## 面试进度轨

聊天面试页展示当前轮次、当前轮提问次数和主题。进度轨按轮推进，不按单个问题推进。

## 信息架构

底部 Tab：

- 首页：开始面试、简历诊断、进行中面试、弱点提醒、最近报告、专项入口、模型配置状态。
- 面试：简历面试、JD 面试、专项面试、恢复进行中面试。
- 历史：报告、弱点、学习状态和复盘对比。
- 我的：模型配置、数据与安全说明。

聊天面试不是底部 Tab，后续作为沉浸式全屏流程进入。

## 模型配置页

模型配置页聚焦 BYOK：

- 选择供应商：OpenAI-compatible、Qwen、GLM、Kimi。
- 输入 API Key。
- 输入 Base URL，可选。
- 选择或填写模型名。
- 测试连接。
- 保存默认模型。

API Key 只进入 `expo-secure-store`。

## 简历诊断页

MVP 优先支持粘贴文本。PDF 和图片入口可以保留，但深度解析后置。

诊断结果需要给出问题、建议、优化示例和推荐面试关注点。

## 聊天面试页

聊天页是核心体验：

- AI 每次只问一个问题。
- 用户回答后才继续。
- 当前轮主题、轮次和提问次数始终可见。
- 请求失败可重试当前步骤。
- 用户可手动结束面试。

## 每轮点评卡片

点评卡片应让用户快速知道：

- 这次回答好在哪里。
- 明确失分点是什么。
- 下一次怎么组织更好。
- 是否会追问，为什么。

## 评估报告页

报告必须绑定证据锚点，避免空泛评价。重点展示总分、维度分、主要短板、高风险问题和下一次练习建议。

## 学习模式页

学习模式根据报告生成学习档案，每次只讲一个知识点、只给一个练习。学习完成状态会回写历史和弱点追踪。

## 历史记录页

历史页展示面试方向、日期、总评分、主题轮数、主要短板和学习状态。无 API Key 时也允许查看本地历史。

## 组件系统

- `Screen`: SafeArea 和页面背景。
- `AppCard`: 支持 `default / info / success / warning / danger` tone。
- `AppButton`: 支持 `primary / secondary`。
- `useTheme`: 统一读取颜色和排版 token。

## 动效原则

- 入口切换轻量、快速。
- 面试进度推进要明确，但不打断输入。
- 点评结果出现时可使用短淡入和轻微位移。
- 动效服务于状态变化，不做装饰性噪音。

## 原型范围

静态 HTML 原型覆盖首页、面试工作台、聊天面试、每轮点评、评估报告和学习模式，同时展示 iPhone 深色和 Android 浅色。
